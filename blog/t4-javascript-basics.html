JavaScript Basics

  • How does JavaScript compare to HTML and CSS?

    • While HTML and CSS let you structure your website and style it respectively, JavaScript adds interactivity and allows to store data. With JavaScript, websites can be responsive to user actions, such as clicking on an element or filling in a form, without having to refresh the entire page. To link a JavaScript file to an HTML file, <script> tag can be used and it is placed at the very end of the <body> element, unlike using <link> tag inside <head> element for linking a CSS file.

  • Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'.

    • Control flow checks to see if the condition is true or false, and then executes a certain code based on the boolean result. Imagine you're a kid. It's dinner time and you need to use the bathroom. Once you're done, your parents might ask "Did you wash your hands?". If you did, you can join them for tea, but if you didn't, they'll say "Please wash your hands", and you'll need to wash your hands.

    Loops are repeating actions carried out until the condition doesn't match. For example, you decided to adopt a dog. Now you've taken the responsibility of taking care of the dog, you need to make sure you feed, walk, wash, and do all the necessary things for your dog while he/she is alive and living with you.

  • Explain the difference between accessing data from arrays and objects.

    • Objects are collections of information that are stored as keys and values in curly braces. Data can be accessed using dot notation or bracket notation along with a key.
    For example:
    var myObject = {name: "Jae", occupation: "web developer"}
    me.name or me["name"] will return "Jae".

    Arrays are ordered collections of data which can be accessed using bracket notation and the index of the desired data.
    For example:
    var myArray = ["Jae", "web developer"]
    myArray[1] will return "web developer".

  • Explain what functions are and why they are useful.

    • Functions are reusable blocks of code. Instead of writing the same or similar code over and over again, we can simply call a function by its name when we need the computer to do the repeating job. It saves a lot of time since programmers don't have to write the same blocks of code throughout their script file. Functions can also take values they use for calculation, called parameters. You can pass different parameters when you call the function for the repeating operation with different values.
