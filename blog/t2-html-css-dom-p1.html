HTML, CSS and DOM - Part 1

  • Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

    • You can think of HTML as the basic skeleton of your car that's basically the same across all the cars of the same model while CSS is like options you can choose such as the colour of your car, the material for the seats or any other modiifcations you make.

    Using HTML, you can include all the contents you'd like to put on a webpage. Then using CSS, you can make the webpage look as you want it like moving around each of the HTML elements or choosing different colours and sizes for them.

    The DOM (Document Object Model) lets the browswer know how to show the HTML document. It organises all the HTML elements in the tree structure allowing the browser to represent them as we want it.

  • What is meant by boxifying design?

    • Boxifying design means dividing all the HTML elements in boxes. You can starting boxifying your design by putting the whole page in a big box, then break down each part of the page such as header, content and footer into smaller boxes making your way into each HTML element.

  • What is the box model?

    • The box model includes four parts: content, padding, border and margin. The content is what you have in the corresponding element in your HTML document, and padding adds space around the content leaving a gap between the content and the border. The border is the outline of the content and paddings, and margin wraps around the border and keeps a space between boxes.

    With all the padding, border and margin taking customised values, it can be confusing to figure out the actual box size. To figure out the size of an element, you need to add the width or height of content, paddings and border. But if you use "box-sizing" property with the value "border-box" in your CSS file, it makes sure the box size doesn't change even after border and padding values change.
